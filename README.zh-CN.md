## 这是什么？

**XIA Sandbox 让*真正的* Linux 程序 —— 真 Python、真 Shell、真 SQLite —— 完全运行在浏览器标签页里。**

不是重新实现，不是租来的云虚拟机。真正的二进制运行在用户自己的设备上，纯 WebAssembly，**任何数据都不上传服务器**。把 SDK 放进任意 Web 应用，就能给用户一个秒级启动的完整计算环境。

它的原理是**把 x86 机器码提前（AOT）翻译成 WebAssembly** —— 既有真实工具链的保真度，又有 Web 平台的安全性与可达性。

## 你能用它构建什么

- **隐私优先 / 本地优先的应用** —— 处理绝不能上服务器的文档、数据或密钥。
- **浏览器内数据科学** —— 用 NumPy / pandas / Matplotlib 做笔记本与分析，无需托管内核。
- **文档自动化** —— 客户端读写 Word、Excel、PowerPoint、PDF。
- **安全的 AI Agent 执行** —— 让 Agent 在用户自己掌控的沙箱里跑真实代码。
- **交互式教学与文档** —— 用真解释器跑可运行示例，零安装。
- **边缘 & 离线工具** —— 断网也能继续计算。

## 工作原理

*（高层视角 —— 不含代码，只讲思路。）*

```
   你的 Web 应用
        │  即插即用 SDK
        ▼
  ┌──────────────────────────────────────────────┐
  │  XIA Sandbox  （运行在 Web Worker 中）          │
  │                                                │
  │   真实 Linux 程序 (ELF)                         │
  │        │  提前 (AOT) 翻译                       │
  │        ▼                                        │
  │   WebAssembly  ──►  在设备上执行                 │
  │                                                │
  │   • 多运行时，自动路由                           │
  │   • 持久化文件系统                              │
  │   • 内容寻址、缓存、带版本的资源                  │
  └──────────────────────────────────────────────┘
        │
        ▼
   结果留在设备上
```

这张图里没有服务器 —— 而这正是重点。
